# AX 세 번째 글 — 페타바이트와 피지컬 AI

> 이 파일은 사용자가 자료·메모·생각을 자유롭게 적는 자리.
> 형식 자유. Claude는 글 작성 시 이 폴더 안 모든 파일을 함께 읽는다.

## 주제 가설 (사용자가 수정·확장)

피지컬 AI(자율주행·로봇·드론·산업용 매니퓰레이터 등)는 학습·검증에 사용하는 센서 데이터가
이전 세대의 AI 학습 데이터와 단위가 다르다. 텍스트 기반 LLM이 테라바이트 단위에서
훈련됐다면, 피지컬 AI는 차량 1대·로봇 1대가 하루에 만들어내는 raw 센서 로그만으로도
페타바이트로 진입한다. 이 양적 차이가 데이터 파이프라인·저장·라벨링·재현성·평가의
모든 단계를 다시 설계하게 만든다.

## 다룰 후보 축 (자유롭게 채우기)

- [ ] 데이터 규모의 실제 숫자 — 자율주행 차량·휴머노이드 로봇·산업 로봇 한 대당 일일 발생량
- [ ] 센서 종류별 비중 — LiDAR / 카메라(다중) / IMU / 레이더 / 마이크 / 햅틱
- [ ] 페타바이트급 저장·전송 인프라 — 차량 내 엣지 저장, 차고지·공장 다운로드 노드, 클라우드 적재 비용
- [ ] 라벨링·큐레이션의 변화 — 자동 라벨링, 액티브 러닝, 시뮬레이션 보강
- [ ] 시뮬레이션과의 비중 — Isaac Sim·Carla·합성 데이터 비중이 왜 점점 커지는가
- [ ] 평가의 재현성 — 같은 모델을 다시 평가하려면 페타바이트 단위 데이터셋 버전을 어떻게 고정하는가
- [ ] 국내 정책·공모 흐름 — 피지컬 AI 관련 공공 R&D·AX 사업에서 데이터 규모 요건이 어떻게 명시되는가

- 카테고리: **AX** (`ax`)
- 분류: 스터디 노트 (`CATEGORIES.md` § 10 참고)
