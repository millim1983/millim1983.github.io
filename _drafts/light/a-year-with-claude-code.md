## 도구가 옮겨놓은 자리

도구의 역사는 비슷한 일을 반복한다. 사람의 자리를 한 칸 옮겨놓는다.

엑셀 이전에는 표 계산을 수식 푸는 사람이 했다. 엑셀이 보편화된 뒤로 사용자는 함수와 입력 셀만 정해주면 된다. 그때 "표 계산을 할 줄 안다"는 말의 의미는 수식을 푸는 능력에서 **무슨 함수를 어디에 쓸지 정하는 능력**으로 옮겨갔다.

같은 이동이 여러 번 일어났다. 디지털 카메라가 노출과 초점을 자동화한 뒤, 촬영하는 사람의 일은 무엇을 어떻게 담을지 정하는 쪽으로 옮겨갔다. GPS 내비게이션 이후 운전의 무게는 어디로 갈지로 좁혀졌다. 번역 모델이 일정 수준 이상을 해낸 뒤, 외국어 실력의 무게는 단어 암기에서 맥락 판단으로 갔다.

Claude Code도 같은 종류의 일을 한다. 코드를 직접 짜는 능력에서 **무엇을 만들지 정의하고 결과를 검증하는 능력**으로 자리가 옮겨간다. 2023년 OpenAI 공동창업자 Andrej Karpathy가 X에 적은 한 줄이 이 변화를 잘 요약한다.

> "The hottest new programming language is English."
> — Andrej Karpathy, 2023

자연어로 의도를 정확히 전달하고 결과를 정확히 검증할 수 있으면, 도구가 그 사이의 코드를 써 준다.

## 그럼 사람이 하는 건 뭐냐

내가 자리에서 하는 일을 적으면 대체로 이렇다.

- 어떤 자료를 다룰지 정한다
- 어디서 가져올지, 무엇을 보존하고 무엇을 비울지 정한다
- 실패 시 어떻게 회복할지 정한다
- 결과가 맞는지 표본으로 확인한다

코드 작성량으로는 1 %도 안 된다. 그런데 시스템의 성격은 거의 다 여기서 결정된다.





## 그래서 이 블로그는

코드 에이전트와 함께하는 여정을 기록하고,
정보 홍수속에서 그래서 '나'는, '인간'은 앞으로 어디로 흘러가야할지 이정표가 되어줄것이라고 생각한다. 
나의 역량, 인간은 어떤 역량을 길러야 미래에 살아남을수있는가?에 대한 생존전략이다. 
'나'와 그리고 나의 소중한 '다음세대'를 위해서. 그 답을 한 편씩 써 본다.
