## 0. 한 줄 정의

**AJAX(Asynchronous JavaScript And XML)** — 웹페이지가 통째로 새로고침되지 않은 채로, 자바스크립트가 백그라운드에서 서버에 요청을 보내고 응답을 받아 화면 일부만 갱신하는 방식.

이름은 옛날부터 굳어진 거고, 지금은 XML보다 JSON을 더 많이 쓴다. 그래도 AJAX라고 부른다.

## 1. AJAX 이전과 이후

**AJAX 이전**의 웹은 단순했다. 사용자가 어떤 버튼이나 링크를 누르면, 브라우저가 서버에 요청을 보내고, 서버가 새 HTML 페이지를 통째로 돌려준다. 화면이 깜빡이면서 다시 그려진다.

```
[사용자 클릭] → [페이지 전체 요청] → [페이지 전체 응답] → [화면 깜빡 후 새 화면]
```

**AJAX 이후**에는 자바스크립트가 페이지를 새로 받지 않고도 서버와 대화한다. 화면 깜빡임 없이, 필요한 데이터만 받아 와서 화면 일부만 바꾼다.

```
[사용자 클릭] → [JS가 백그라운드 요청] → [JSON 응답] → [JS가 화면 일부만 갱신]
```

검색창에 글자를 칠 때 자동완성 추천이 뜨는 것, 페이지 끝까지 스크롤하면 게시물이 자동으로 더 불러와지는 것, "더보기" 버튼을 눌렀을 때 페이지 전체가 새로 안 뜨고 그 자리에 글이 추가되는 것 — 모두 AJAX 패턴이다.

## 2. 실제로 어떤 일이 일어나는가

브라우저 안의 자바스크립트가 HTTP 요청을 하나 만든다. 이 요청은 사용자 눈에는 보이지 않는다. 옛날 코드에서는 `XMLHttpRequest` 객체를 썼고, 요즘은 `fetch()` 함수를 더 많이 쓴다.

```javascript
fetch('/api/announcements?page=1')
  .then(res => res.json())
  .then(data => {
    // 받은 JSON을 화면에 그리는 코드
  });
```

서버는 평소처럼 HTTP 응답을 돌려준다. 다만 응답이 HTML이 아니라 보통 JSON이다.

```json
{
  "items": [
    {"id": "001", "title": "공고 제목 1"},
    {"id": "002", "title": "공고 제목 2"}
  ],
  "totalCount": 142
}
```

JS는 그 JSON을 받아서 페이지 안의 어딘가에 새 항목을 그려 넣는다.

## 3. 개발자 도구에서 보는 법

이게 코딩이 본업이 아닌 사람이 알면 가장 쓸모 있는 부분이다. **브라우저는 AJAX 호출을 모두 기록하고 있다.**

1. 페이지에서 F12 (또는 우클릭 → 검사) → 개발자 도구
2. 상단 탭에서 **Network** 선택
3. 왼쪽 필터에서 **XHR** 또는 **Fetch/XHR** 선택
4. 페이지에서 버튼을 누르거나 스크롤하면, 그때 발생한 요청이 한 줄씩 추가된다
5. 각 요청을 클릭하면 URL·메서드·요청 본문·응답 본문을 다 볼 수 있다

이걸로 알 수 있는 것:
- 이 사이트가 어떤 데이터를 어디서 어떻게 받아오는가
- 응답이 JSON이라면 그 키가 어떻게 생겼는가
- 같은 데이터를 여러 번 받아오는가, 한 번만 받아 캐시하는가

본 블로그가 시리즈 1편에서 다룬 "데이터 출처의 세 층위"의 가운데 층(AJAX 엔드포인트 직접 호출)이 이 관찰에서 시작한다.

## 4. AJAX가 답이 아닐 때

AJAX는 만능이 아니다. 다음 경우엔 다른 방식이 더 낫다.

- **검색 엔진에 보여야 할 콘텐츠** — AJAX로만 그리면 검색 봇이 못 본다. 서버 사이드 렌더링이 필요하다.
- **첫 화면이 빨라야 할 때** — 첫 진입 시 빈 페이지가 뜨고 JS가 데이터를 받아 그리기 시작하면, 사용자가 빈 화면을 잠시 본다. 첫 화면은 서버가 미리 그려서 보내는 게 빠르다.
- **데이터가 거의 변하지 않을 때** — 정적 HTML 한 장이면 충분한 걸 굳이 AJAX로 매번 받아오는 건 낭비.

## 5. 한 줄 마무리

> **AJAX = 화면 새로고침 없이 서버와 대화하는 방식.** 자바스크립트가 백그라운드로 요청을 보내고, 보통 JSON을 받아 화면 일부만 바꾼다.

개발자 도구의 Network 탭에서 모든 AJAX 호출을 그대로 들여다볼 수 있다는 사실이 가장 쓸모 있는 한 가지다. 이걸 알면 웹페이지의 동작을 한 단계 깊게 읽을 수 있게 된다.
