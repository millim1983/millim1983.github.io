## 0. 숫자가 알려주는 일

서버에 무언가 요청했을 때 돌아오는 숫자 3자리. 이 숫자가 **누가 잘못했는지, 다음에 무엇을 해야 하는지**를 알려준다. 자주 만나는 7개만 한 줄씩 정리한다. 코딩이 본업이 아닌 사람도 이 7개만 알면 90 %의 상황을 읽을 수 있다.

## 1. 첫 자리로 종류를 가른다

| 첫 자리 | 의미 | 한 줄 |
|---|---|---|
| 1xx | 정보 | 자주 안 본다 |
| 2xx | **성공** | 잘 됐다 |
| 3xx | **리다이렉트** | 다른 데로 가라 |
| 4xx | **클라이언트(나) 잘못** | 내 요청에 문제 있다 |
| 5xx | **서버 잘못** | 서버가 죽었거나 버그다 |

첫 자리만 봐도 "이건 내가 고칠 일이냐 서버가 고칠 일이냐"가 갈린다.

## 2. 자주 만나는 7개

### 200 OK — 정상

요청 성공. 응답 본문(JSON·HTML·파일)이 같이 온다. **단, 본문이 비어 있거나 이상한데 200이 오는 경우가 있다.** 응답 본문을 한 번씩 보고 진짜 원하는 데이터가 들어 있는지 확인하는 습관이 필요하다.

### 301 / 302 — 리다이렉트

301은 영구 이동, 302는 일시 이동. `Location:` 헤더에 새 URL이 들어 있다. `requests` 라이브러리 같은 도구는 기본적으로 자동 추적한다. 자동 추적이 의도와 다르면 `allow_redirects=False`로 끈다.

### 401 Unauthorized — 너 누군지 모르겠다

**인증이 필요한데 안 했거나, 한 인증이 잘못됐다.** API 키·토큰·세션 쿠키가 빠졌거나 만료됐다. 확인 순서:
1. 인증 헤더(`Authorization: Bearer ...`)가 실제로 붙어 있는지
2. 토큰이 만료되지 않았는지
3. 환경변수에서 키를 제대로 읽고 있는지 (오타 한 글자에 401이 뜬다)

### 403 Forbidden — 누군지는 알겠는데 권한 없다

인증은 됐는데 그 자원에 접근할 권한이 없다. 401과는 다르다. **권한 정책·요금제·IP 제한·국가 제한** 같은 게 원인. 401은 "다시 로그인하라"의 신호, 403은 "이 자원은 너에게 안 열려 있다"의 신호.

### 404 Not Found — 그런 자원 없다

URL이 잘못됐거나, 자원이 삭제됐거나, 아예 처음부터 없었거나. **"진짜 없는 것"과 "있는데 권한이 없어서 없는 척하는 것"이 같은 404로 표현될 수 있다.** 보안상 일부 서비스는 권한 없는 자원을 403 대신 404로 반환한다.

### 429 Too Many Requests — 천천히 좀

**rate limit에 걸렸다.** 일정 시간 안에 너무 많이 요청했다. 응답에 보통 `Retry-After` 헤더가 같이 온다 — N초 또는 절대 시각. 자동화 코드를 짠다면 다음 두 가지를 같이 설계해야 한다.

- 요청 간 sleep 또는 토큰 버킷
- 429를 만나면 즉시 멈추고 `Retry-After` 만큼 대기 후 재시도

OpenAPI 호출에서 일일 한도가 따로 있는 경우엔 그날 작업을 chunk 단위로 끊고 다음 날 재개하는 방식이 더 안정적이다.

### 500 Internal Server Error — 서버가 죽었다

**서버가 자기 코드에서 예외를 던졌거나, 의존하는 서비스가 죽었다.** 내 잘못이 아닐 가능성이 높다. 하지만 내 요청이 비정상적이라서 서버가 처리를 못 한 경우도 있다. 같은 요청을 잠시 뒤에 다시 보내 보고:
- 같은 500이면 → 서버 문제, 시간 두고 재시도 또는 보고
- 다른 응답이면 → 일시 장애였거나 내 요청이 경계 조건이었던 것

500이 반복되면 그 사이트의 status 페이지를 한 번 확인한다.

## 3. 만났을 때 무엇을 하는가 — 한 줄 정리

| 코드 | 다음 행동 |
|---|---|
| 200 | 응답 본문도 같이 확인 |
| 301/302 | `Location` 보고 의도된 리다이렉트인지 |
| 401 | 인증 헤더·키·만료 점검 |
| 403 | 권한·요금제·IP/국가 제한 확인 |
| 404 | URL·자원 ID 재확인. 권한 가능성도 의심 |
| 429 | `Retry-After` 만큼 대기 → 자동화엔 sleep 추가 |
| 500 | 잠시 후 재시도. 반복되면 서버 status 확인 |

## 4. 한 줄 마무리

상태 코드는 서버가 보내는 **짧은 안내문**이다. 본문을 길게 읽기 전에 이 3자리 숫자부터 보면 다음 행동의 절반이 결정된다. 7개만 외워 두면 충분하다.
